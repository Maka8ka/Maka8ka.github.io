Docker常用命令

关键词: 容器, 运维 状态: 需补充

Docker 相关命令

Docker 示例

  • docker运行MySQL容器

docker run -d -p 3306:3306 —name mysql -e MYSQL_ROOT_PASSWORD=root mysql

docker 运行tomcat 容器 //将宿主机中的文件复制到容器中

docker run -d -p 8080:8080 —name tomcat

docker cp /home/user/web.war 容器id:/usr/local/tomcat/webapps/

  • docker 映射数据卷运行
  • 方法一（常用）：docker run —d v /home/user/web:/usr/local/tomcat/webapps/web -p 8080:8080 —name tomcat
  • 方法二：

docker volume create 数据卷名称

docker run —d v 数据卷名称:/usr/local/tomcat/webapps/web -p 8080:8080 —name tomcat

Docker-Compose

//添加docker参数配置，可批量管理容器

需要从github下载Docker-Compose

下载linux可执行文件移动到/usr/local/bin

添加环境变量/etc/profile

export PATH=/usr/local/bin:$PATH

基于docker-compose启动

docker-compose up -d #在当前路径检测docker-compose.yml

docker-compose down # 相当于stop 和remove

docker-compose start/stop/restart 启动或停止由docker-compose管理的容器

docker-compose ps 查看由docker-compose管理的容器

说明：docker-compose.yml文件已key:value方式管理。多个配置文件以换行+缩进的方式来区分,不要使用制表符tab。

yml文件

version: ‘3.1’
services:
  mysql:   #服务名称
    restart: always    #docekr启动自动启动镜像
    image: mysql   #指定镜像路径
    container_name: mysql  #指定容器名称
    ports:
      - 3306:3306
      - 3316:3316
    environment: //具体配置启动参数可看docker仓库镜像介绍
      MYSQL_ROOT_PASSWORD: root
      TZ: Asia/Shanghai
    volumes:
      - /home/user/web:/usr/local/tomcat/webapps/web #映射数据卷

Docker操作命令

运行容器

docker run 镜像id/镜像名称[:tag] //tag可选操作

docker run -d -p 宿主机端口:容器端口 —name 容器名称 镜像id/镜像名称[:tag]

  • d 后台运行容器
  • p 映射端口

—name 指定容器名称

可以直接run 会自动pull 拉取

停止容器

docker stop 容器id

docker stop $(docker ps -qa) 停止所有容器

进入容器内部

docker exec -it 容器id bash

删除容器（）

docker rm 镜像id

docker rm $(docker ps -qa) 删除所有容器

查看正在运行的容器

docker ps [-q/a]

  • q 只查看id标识
  • a 查看所有容器包括未再运行的容器

查看docker日志

docker logs -f 容器id

Docker 镜像操作

拉取镜像

docker pull nginx

查看本地镜像

docker images

删除本地镜像

docker rmi bf756fb1ae65(镜像id，不需要完整，只保证唯一即可)

镜像的导入/导出/改名

docker save -o ./test.imge 镜像id

docker load -i ./test.imge

docker tag bf756fb1ae65 repository:tag //修改镜像名称

Docker 数据卷**/**文件目录 ****映射

创建数据卷

docker volume create 数据卷名称

创建数据卷之后，默认会存放在一个目录下/var/lib/docker/volumes/数据卷名称/_data

查看数据卷信息

docker volume ls //查看所有数据卷

docker volume inspect 数据卷名称  //查看数据卷信息

docker volume remove 数据卷名称 //删除数据卷信息

Docker 制作镜像

第一步：创建Dockerfile文件

from: 指定以来环境 如 niginx

copy: /home/user/web.war /usr/local/tomcat/webapps

workdir: 声明工作目录（可选）

cmd: 执行的命令（可选）

第二步：执行相应的命令

docker build -t 镜像名称:[tag] . /Dockerfile

第三步：查看镜像（可选）

Docker 宿主 ****配置相关命令

启动docker

systemctl start docker

启动docker

systemctl stop docker

设置docker开机自启动

systemctl enable docker

禁止开启自启动

systemctl disenable docker

docker 修改源当前命令有效

docker run hello-world –registry-mirror=https://docker.mirrors.ustc.edu.cn

docker 修改镜像仓库配置

sudo vim /etc/docker/daemon.json

{
  "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
}

使配置文件生效

sudo systemctl daemon-reload

重启docker

sudo service docker restart

测试是否生效

docker search nginx
